## 核心功能

### 🚀 服务端存储架构

**新增 Node.js + Express 后端服务**

- **架构：** 前端 (Vite/React) → 后端 API (Node.js/Express) → JSON 文件存储
- **端口：** 后端 `:3001` / 前端 `:5173`
- **数据文件：** `server/data/timebank-data.json`
- **优势：**
  - ✅ 数据持久化到服务器，不怕浏览器缓存
  - ✅ 多设备同步（家庭 LAN 内所有设备访问同一份数据）
  - ✅ 每日自动重置（跨天自动清零 `dailyCount`）
  - ✅ 时区正确（北京时间 UTC+8）

### 📡 REST API 接口

| 接口 | 方法 | 功能 |
|------|------|------|
| `/api/data` | GET | 获取完整数据（含自动每日重置） |
| `/api/data` | POST | 保存完整数据 |
| `/api/earn` | POST | 完成任务（含衰减计算） |
| `/api/redeem` | POST | 兑换积分 |
| `/api/timers/clear` | POST | 清除已过期计时器 |
| `/api/timers/:id/pause` | POST | 暂停计时器 |
| `/api/timers/:id/resume` | POST | 继续计时器 |
| `/api/tasks/:id` | PUT | 更新任务配置 |
| `/api/tasks` | POST | 添加任务 |
| `/api/tasks/:id` | DELETE | 删除任务 |
| `/api/balance/adjust` | POST | 调整余额（支持小数） |

### 🔄 每日自动重置

- **触发时机：** 每次获取数据时自动检测跨天
- **重置内容：** 所有任务的 `dailyCount` 归零，`lastUpdate` 更新为今日
- **时区：** 本地时间（北京时间 UTC+8）

## 重要修复

### 🐛 每日计数重置 Bug

**问题：** 每日计数不会在跨天后自动重置

**原因：**
1. `checkDailyReset()` 只在 App 首次加载时执行
2. zustand persist 数据恢复是异步的，重置可能发生在数据恢复之前

**修复方案：**
- 服务端在 `getData` 时自动检测并重置
- 前端每小时自动刷新数据
- 时区修正为本地时间（北京时间）

### 🐛 输入框失焦问题

**问题：** 编辑任务配置时，每次输入都会触发保存导致失焦

**修复：**
- 添加本地状态（`editingDesc`、`editingPoints`）
- 输入时只更新本地状态
- 失去焦点时才调用 API 保存

### 🐛 UI 布局问题

**问题：**
1. Tab 栏滚动时被内容覆盖
2. 输入框过长遮挡右侧按钮

**修复：**
- Tab 栏 z-index 从 10 提升到 50
- 输入框改用 flex 布局（`flex-[2]` + `flex-[1]`）
- 按钮添加 `whitespace-nowrap` 和 `max-w-fit`
